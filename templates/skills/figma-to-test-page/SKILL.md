## 目标
把 Figma 设计稿快速落地成“可验证的测试页”（test page）：页面结构、视觉样式、关键交互尽量还原，并可在本地/预览环境直接打开检查。

## 输入（你需要提供）
- Figma 链接或关键信息（页面名、Frame 名、组件名）
- 目标技术栈约束（React/Vue/Nuxt/Next/纯静态等）
- 设计还原优先级
  - 必须还原：布局/字体/间距/颜色/圆角/阴影/响应式
  - 可放宽：动效/复杂图片资源/特定字体
- 资源约束：是否允许下载图片/SVG、是否已有设计 token

## 输出（我会交付）
- 一个独立可访问的测试页（建议路径示例）
  - `src/pages/__test__/figma/{{SKILL_SLUG}}.vue` 或
  - `src/pages/__test__/figma/{{SKILL_SLUG}}.tsx`
- 样式与结构拆分（尽量组件化，避免巨型单文件）
- 必要的 mock 数据与占位资源
- 简短验收清单（逐项对照 Figma）

## 工作流程
- 从 Figma 提取结构：大区块 -> 组件 -> 细节
- 先搭骨架与网格/布局（flex/grid），保证对齐与间距
- 再补视觉：字号、行高、颜色、背景、圆角、阴影
- 最后补交互：hover/active/focus、滚动、溢出、省略
- 产出验收清单，便于你快速对比截图/像素差异

## 约束与约定
- 测试页优先“可比较、可回归”，不追求一次性完美抽象
- 不随意引入新依赖；如需引入（例如图标/样式工具）会说明
- 资源下载受限时：用占位图/渐变块/SVG 占位并留 TODO

## 验收清单（示例）
- 布局：容器宽度、左右边距、栅格列数、对齐方式一致
- 文本：字体、字号、行高、字重、颜色一致；溢出规则一致
- 组件：按钮/输入框/卡片状态（default/hover/disabled）一致
- 响应式：在 1440/1024/768/375 下不破版
